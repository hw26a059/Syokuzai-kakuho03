# 食材目利きゲーム 仕様書：ゲームシステム

## 1. クイズシステム（目利き）
- **出題カテゴリ**: お肉、お魚、野菜
- **出題内容**:
  - 食材の鮮度の見分け方（色、つや、ドリップの有無など）
  - 食材の旬の時期や、旬のものの特徴（例：春キャベツと冬キャベツの違いなど）
- **出題数**: 
  - プレイヤーが選択した「料理レシピ」に使う食材の数（「具材」）に応じて変動。
  - 例：カレーライスで「じゃがいも」「にんじん」「玉ねぎ」「牛肉」を選択した場合、問題数は4問（※具材のカスタマイズ状況によって基本食材＋トッピング具材はクイズ対象から除外されることがあります）。
- **出題順**:
  - プレイヤーがカスタマイズした食材の並び（または定義順）に基づき、肉・魚・野菜といったカテゴリを跨いで連続で出題。
- **難易度設定とゲームルール**:
  - クイズ開始前に「簡単（標準）」と「難しい」の2つの難易度から選択できる。デフォルトは「簡単（標準）」が選択。
- **制限時間**:
  - **簡単（標準）**: 1問あたり60秒。時間に追われすぎず、じっくり考えて回答可能。
  - **難しい**: 1問あたり30秒。瞬時の判断力が必要。
- **回答方式と難易度別の挙動**:
  - **簡単（標準）**:
    - 写真付き問題（豚肉・鶏肉）：選択肢に「状態説明文」と「詳細テキスト」が併記され、写真付き（3択、素材数が限られる場合は2択）で出題。
    - テキスト問題：写真アセットのない問題はテキストのみの3択形式で出題。
  - **難しい**:
    - 写真付き問題（豚肉・鶏肉）：ヒントとなるテキスト（状態説明や文章）が完全に非表示になり、**「写真（画像アセット）のみ」**からプレイヤー自身の目で直接目利きを行う（2択）。
    - テキスト問題：選択肢がより紛らわしい表現に差し替わり、さらに**選択肢が1つ増えた「4択形式」**（4択）で出題。
- **写真の選択肢構成**:
  - 最も新鮮・旬の状態（正解）、少し時間が経過してドリップ等が出ている状態（不正解）、変色や傷みが見られる状態（不正解）などで構成。
- **ミニ解説機能**: クイズ回答後、すぐにその食材の選び方（目利き方法、保存方法など）を解説する画面を表示。

## 2. 料理システム
- **レシピの選択と食材カスタマイズ**:
  - プレイヤーは最初に作りたい「料理レシピ」を選択する。
  - レシピ選択後、チェックボックス等で必須食材に加え、具材の変更・追加（例：しめじ、茄子、蓮根、豚肉、鶏肉、イカ、海老など）を自由に行える。
  - ただし、明らかにその料理に合わない不味い組み合わせ（例：麻婆豆腐にバナナなど）は、選択肢からあらかじめ除外される。
- **料理の評価と結果への影響**:
  - **美味しさ判定（失敗基準）**:
    - クイズの「正解率が50%以上（例：4つの食材中2問以上正解）」であれば、美味しい料理が完成。
    - 正解率が50%未満（例：4つの食材中3問以上不正解）の場合、料理は「不味く（失敗扱い）」なる。
- **自由命名（保存・図鑑登録）**:
  - 無事に美味しい料理が完成した際、プレイヤー自身が*料理名を自由に変更（命名）*して保存できる。
  - 例：ベースレシピが「カレーライス」で、具材をイカとエビに変更した場合、プレイヤーが自ら「シーフードカレー」と命名して図鑑に登録できる。
- **料理図鑑（コレクション表示）**:
  - 美味しく完成した料理（プレイヤーが命名した名前と、使用した食材データなど）は図鑑にコレクションされる。クイズに間違えて不味くなった（失敗した）料理は、図鑑には登録されない。
  - 図鑑内では、ベースとなった「レシピ（タグ）」（例：「カレーライス」「肉じゃが」など）ごとにグループ分けされ、整理して表示される。

## 3. スコア・ランキングシステム
- **ニックネーム・ユーザー管理**:
  - プレイヤーは初回起動時（または名前未登録時）に「ニックネーム」を入力・登録（簡易ログイン）する。
  - プレイヤー固有のニックネームと、これまでに獲得した「累計スコア」が、クラウド（Firestore等）上で永続的に管理される。
- **スコア加算機能**: 良い食材（クイズ正解）を選ぶごとにスコアが加算される。
- **ランキング機能**:
  - プレイヤー同士が、これまでに獲得した「累計スコア」の高さを競い合う。
  - ランキングボードには、システム上の**上位50名**のスコアとニックネームが表示される。
